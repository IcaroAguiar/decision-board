# Agentic Quality Gate Ratchet

This package was generated by `agentic-code-review/scripts/generate-quality-gate-ratchet.mjs`.

## Policy

- Keep new code clean first; do not require a big-bang cleanup of legacy code.
- Every PR publishes floating metrics: baseline, current value, target, delta, and status.
- Block regressions: high/blocking review findings, secret leaks, failing native checks, coverage drops, duplication increases, and large-file growth.
- Touched bad areas must improve or stay flat. If a PR touches an already-large file, the file must not grow without explicit review policy change.
- Baseline refresh is explicit and only allowed from a passing default-branch quality gate report.
- False positives and false negatives are tracked in `review-feedback.json`; false negatives produce an auto-improvement queue.
- Useful comments explain invariants, business rules, edge cases, and agent navigation context. Narrating syntax does not count as maintainability.

## Files

- `baseline.json`: current accepted state.
- `quality-gate.config.json`: thresholds and required checks.
- `check-ratchet.mjs`: CI checker and PR report generator.
- `refresh-baseline.mjs`: controlled baseline refresh after a passing gate.
- `review-feedback.json`: optional human/reviewer feedback for false positives and false negatives.
- `github-action-quality-gate-ratchet.yml`: workflow template; copy or merge into `.github/workflows/quality-gate.yml`.
- `vendor/collect-review-context.mjs`: vendored deterministic collector used by CI because GitHub Actions cannot assume `~/.agents` exists.

The active GitHub Actions workflow lives at `.github/workflows/quality-gate.yml`.

The deterministic collector ignores this generated gate package and its own
GitHub Actions workflow through `.agentic-reviewrc.json`. Changes to the gate
itself must be reviewed explicitly with an independent reviewer because the
gate should evaluate product/application PRs, not fail on its own vendored
implementation details. The workflow still runs `node --check` against the
gate scripts so syntax errors fail before merge.

## Local use

Run the same ratchet locally before opening a PR:

```bash
node docs/ai/quality-gate/vendor/collect-review-context.mjs --base origin/main --json > docs/ai/quality-gate/agentic-review-packet.json
node docs/ai/quality-gate/check-ratchet.mjs
```

This writes ignored local artifacts:

```txt
docs/ai/quality-gate/agentic-review-packet.json
docs/ai/quality-gate/quality-gate-report.json
docs/ai/quality-gate/quality-gate-report.md
docs/ai/quality-gate/quality-trend-entry.json
docs/ai/quality-gate/auto-improvement-queue.json
```

Manual baseline refresh writes the updated baseline in the workflow checkout.
The workflow uploads `baseline.json` and `quality-trend.jsonl` as artifacts so
the baseline can be reviewed and applied through an explicit follow-up PR.

## Feedback schema

`review-feedback.json` accepts:

```json
{
  "feedback": [
    {
      "case": "pr-123",
      "rule": "webhook-without-signature-verification",
      "outcome": "false-negative",
      "recommendation": "Add a detector for provider secret-token validation gaps."
    }
  ]
}
```

Use `false-positive`, `false-negative`, or `severity-mismatch` as outcomes.
